# Design System & Guía de Estilo

## JSoft Solutions

> Documento operativo en **Markdown** para uso directo en el proyecto y como referencia semántica en el IDE.
> Esta guía define **identidad, estilo visual, jerarquía UI y criterios de profesionalización** del sitio web.

---

## 1. Propósito del Documento

Este archivo es la **fuente única de verdad** para:

* Decisiones de diseño visual
* Estilización CSS / Tailwind
* Composición de layouts y componentes
* Consistencia semántica de UI
* Profesionalización estética sin recurrir a efectos decorativos

El objetivo no es “verse bonito”, sino **verse como un sistema de ingeniería confiable**.

---

## 2. Posicionamiento de Marca

**Tipo de marca:** Ingeniería de software aplicada
**Percepción buscada:** Técnica, confiable, estructurada, escalable
**Promesa implícita:** Soluciones sólidas, seguras y bien diseñadas

La marca **no comunica creatividad artística**. Comunica:

* Arquitectura
* Proceso
* Método
* Previsibilidad
* Control técnico

---

## 3. Núcleo Simbólico

### 3.1 Montaña — Arquitectura y Escalabilidad

La montaña representa:

* Fundamentos sólidos
* Crecimiento progresivo
* Resiliencia técnica
* Visión de largo plazo
* Estabilidad estructural

No es un recurso decorativo. Es una **metáfora directa de arquitectura de software**.

---

### 3.2 Monograma JS Integrado

Principio:

* El **JS no flota ni se superpone**
* Está integrado en la estructura
* El stack es base, no accesorio

Simboliza:

* Dominio técnico
* Especialización
* Coherencia arquitectónica

---

### 3.3 Code Paths — Flujo de Datos

Los trazos tipo circuito representan:

* Data flow
* Pipelines
* Backend → Frontend
* Procesos
* Trazabilidad

Regla visual:

* Dirigidos
* Geométricos
* Ordenados
* Controlados

Nunca caóticos.

---

## 4. Lenguaje Cromático (Semántico)

> El color **no es estético**, es semántico.

### 4.1 Azul — Frontend / Claridad / Confianza

Representa:

* Interfaz
* Visibilidad
* UX
* Confianza

Debe dominar:

* Fondos
* Capas principales
* Contexto visual

---

### 4.2 Verde — Backend / Seguridad / Ejecución

Representa:

* Runtime
* Validación
* Seguridad
* Integridad de datos

Uso **restringido** a señales técnicas.

---

### 4.3 Blanco / Gris Frío — Ingeniería

Representa:

* Claridad
* Precisión
* Documentación
* Legibilidad

Se usa para:

* Respiración visual
* Separación de bloques
* Jerarquía tipográfica

## 5. Paleta Oficial

### 5.1 Azules — Núcleo de Marca

| Rol                | Color     | Uso                          |
| ------------------ | --------- | ---------------------------- |
| Azul Base Profundo | `#192950` | Fondos, hero, header, footer |
| Azul Medio Técnico | `#21497B` | Secciones, cards, paneles    |
| Azul Activo UI     | `#25609D` | Botones primarios, links     |

---

### 5.2 Verdes — Señal Técnica

| Rol           | Color     | Uso                               |
| ------------- | --------- | --------------------------------- |
| Verde Backend | `#3E985D` | Badges, indicadores técnicos      |
| Verde Acento  | `#7CBD68` | Confirmaciones, estados positivos |

---

## 6. Proporción de Uso

* Azules: **75–85%**
* Verdes: **10–18%**
* Neutros: resto

> Si el verde supera ~20%, deja de ser señal y se vuelve ruido.

---

## 7. Reglas de Coherencia Visual

### Regla 1 — Un rol, un color

No intercambiar funciones semánticas.

* Botón primario → Azul Activo
* Badge técnico → Verde Backend
* Confirmación → Verde Acento

---

### Regla 2 — No inventar variaciones

Prohibido:

* Azules “parecidos”
* Verdes ajustados
* Degradados improvisados

Solo colores oficiales o transparencias.

---

### Regla 3 — Consistencia por componente

Un componente define su color **una vez** y se respeta en todo el sistema.

---

## 8. Profesionalización del Estilo (Clave)

> Aquí se eleva el diseño sin recurrir a efectos.

### 8.1 Separación Visual

Usar:

* Espaciado generoso vertical
* Bloques claramente delimitados
* Ritmo visual consistente

Evitar bloques densos sin respiración.

---

### 8.2 Capas de Azul (Profesionalización)

No todo debe ser el mismo azul.

Patrón recomendado:

* Fondo general → Azul Base
* Secciones → Azul Medio
* Contenido destacado → Azul Medio con borde sutil
* Interacción → Azul Activo

Esto crea **profundidad estructural**, no decorativa.

---

### 8.3 Bordes y Límites

Preferir:

* Bordes sutiles
* Líneas finas
* Separadores discretos

En lugar de:

* Sombras pesadas
* Glow
* Overlays

---

### 8.4 Tipografía y Legibilidad

Principios:

* Jerarquía clara de títulos
* Longitud de línea controlada
* Contraste suficiente

El texto debe ser **escaneable**, no decorativo.

---

## 9. Seguridad como Capa Arquitectónica

La seguridad se representa mediante:

* Mallas
* Capas
* Topologías

Nunca mediante:

* Candados grandes
* Iconos genéricos
* Estética hacker

La seguridad **se siente**, no se grita.

---

## 10. Estilo de Composición

### Geometría

* Líneas rectas
* Ángulos definidos
* Simetría controlada
* Grids implícitos

### Densidad

> Siempre debe haber más estructura que efecto.

## 11. Coherencia con el Perfil Técnico

La marca comunica:

* Arquitectura
* Procesos

* Seguridad progresiva

Una vista cumple esta guía si:

* Predomina el azul
* El verde dirige la atención
* No hay colores fuera de la paleta
* La jerarquía se entiende sin texto
* Parece un sistema, no decoración
